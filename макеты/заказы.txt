# Экран «Заказы» — подробное описание

Экран «Заказы» является центральным рабочим инструментом для сотрудников «Мастер Принт». Здесь происходит вся операционная деятельность: регистрация новых обращений, отслеживание хода ремонта, изменение статусов, фиксация использованных материалов и завершение заказов.

---

## 1. Общая структура экрана

Экран состоит из трёх основных частей:
- **Список заказов** (таблица с фильтрами и поиском)
- **Форма создания / редактирования заказа** (открывается по кнопке или при выборе заказа)
- **Карточка детального просмотра заказа** (открывается при клике на строку списка или по кнопке «Подробнее»)

Доступ к функциям зависит от роли пользователя (администратор, менеджер, мастер). Ниже описан полный функционал для администратора; для других ролей указаны ограничения.

---

## 2. Список заказов

### 2.1 Внешний вид
- Таблица с колонками (настраиваемыми: администратор может выбрать, какие колонки отображать):
  - **Номер заказа** (формат `MMDD-XXXX`)
  - **Дата приёма**
  - **Клиент** (ФИО или название организации)
  - **Устройство** (тип + модель)
  - **Неисправность** (кратко, первые 40 символов)
  - **Статус** (цветной индикатор + текст)
  - **Ответственный мастер**
  - **Предварительная стоимость** / **Окончательная стоимость**
  - **Дата готовности** (желаемая / фактическая)
  - **Действия** (иконки просмотра, редактирования — если разрешено)

- Пагинация: 20, 50, 100 записей на страницу.
- Сортировка: по любому столбцу (по умолчанию — по дате приёма, новые сверху).
- Фильтры над таблицей (с возможностью сохранить комбинацию фильтров как «быстрый фильтр»):
  - **Статус** (мультивыбор, счётчики количества заказов по каждому статусу)
  - **Диапазон дат приёма** (календарь «с» и «по»)
  - **Клиент** (поиск по имени/телефону с автодополнением)
  - **Мастер** (выпадающий список активных мастеров)
  - **Поисковая строка** (по номеру заказа, телефону клиента, устройству, модели, серийному номеру)
  - **Кнопка «Сбросить фильтры»**

### 2.2 Быстрые действия в списке
- Иконка «Редактировать» (для администратора и менеджера — переход к форме редактирования)
- Иконка «Статус» (быстрая смена статуса без открытия карточки — выпадающий список доступных статусов)
- Для заказов со статусом «Готов» может быть кнопка «Выдать» (переводит в «Выдан» одним кликом)

---

## 3. Создание нового заказа

### 3.1 Открытие формы
- По кнопке «+ Создать заказ» над таблицей.
- При создании из карточки клиента — форма предзаполняется данными клиента.

### 3.2 Поля формы (группированы)

#### Блок 1: Клиент
- **Клиент** — поле с автодополнением по существующим клиентам (по имени, телефону). Если клиент не найден, рядом кнопка «+ Новый клиент», которая открывает модальное окно для быстрого добавления (обязательные поля: имя, телефон, опционально email, адрес). После создания клиент автоматически подставляется в форму.

#### Блок 2: Устройство
- **Тип устройства** (выпадающий список: Принтер / МФУ / Ноутбук / Монитор / Другое) — влияет на предлагаемые модели.
- **Модель** — текстовое поле с подсказками на основе ранее введённых моделей.
- **Серийный номер** — текстовое поле (опционально).
- **Неисправность** — многострочное поле (обязательное). Можно использовать быстрые шаблоны (кнопки: «Не печатает», «Шумит», «Не включается»), которые вставляют текст.

#### Блок 3: Комплектация (чекбоксы)
- Блок питания
- Кабель USB
- Кабель питания
- Мышь
- Клавиатура
- Картридж
- Поле «Другое» (текстовое)

#### Блок 4: Даты и стоимость
- **Дата приёма** (календарь, по умолчанию сегодня)
- **Желаемая дата готовности** (календарь, опционально)
- **Предварительная стоимость** (числовое поле, опционально)
- **Окончательная стоимость** (будет заполнена позже, при закрытии заказа)

#### Блок 5: Фото
- Область загрузки фото (drag & drop или выбор файлов). Можно прикрепить до 3 изображений. Фото хранятся на сервере и привязываются к заказу. Миниатюры отображаются в форме.

#### Блок 6: Ответственный
- **Принявший сотрудник** (выпадающий список мастеров и менеджеров, по умолчанию текущий пользователь)
- **Назначенный мастер** (для администратора — можно выбрать другого мастера, для менеджера — только выбор из списка мастеров, для мастера — только он сам)

#### Блок 7: Дополнительно
- **Комментарий** (внутренняя заметка, видна только сотрудникам)

### 3.3 Кнопки формы
- **Сохранить** — создаёт заказ со статусом «Принят», присваивает номер заказа (формат `MMDD-XXXX`), отправляет SMS-уведомление клиенту (если настроено).
- **Сохранить и продолжить** — после сохранения открывается карточка заказа.
- **Отмена** — возврат к списку.

---

## 4. Карточка заказа (детальный просмотр)

Открывается при клике на строку заказа в списке или по кнопке «Подробнее». Содержит всю информацию о заказе, историю изменений, использованные материалы и возможность выполнения действий.

### 4.1 Верхняя часть
- Номер заказа, статус (крупно, с цветным индикатором)
- Кнопки: **Редактировать** (форма), **Печать** (распечатка карточки заказа), **История** (лента изменений)

### 4.2 Основные вкладки

#### Вкладка «Детали заказа»
- Все поля из формы создания (клиент, устройство, неисправность, комплектация, даты, стоимость, фото, ответственные). Поля доступны для редактирования в зависимости от роли.
- При изменении любого поля сохраняется история изменений (кто, когда, что изменил).

#### Вкладка «Статусы и история»
- Лента событий (хронологический список):
  - Смена статуса (дата, время, кто изменил, комментарий)
  - Отправка SMS-уведомлений клиенту (с указанием текста и времени)
  - Добавление / удаление материалов
- Внизу — поле для добавления комментария к текущему изменению статуса.

#### Вкладка «Материалы»
- Таблица использованных материалов:
  - Наименование, артикул, количество, цена за единицу (если ведётся учёт), сумма
- Кнопка «Добавить материал» (для мастера и администратора):
  - Открывает окно выбора товара из склада (поиск по названию/артикулу)
  - Поле для ввода количества (с проверкой наличия)
  - После добавления материал автоматически списывается со склада (создаётся запись в `stock_movements` с привязкой к заказу)
- Кнопка «Удалить» (рядом с каждой позицией) — возвращает материал на склад и удаляет запись (доступно только администратору)

#### Вкладка «Уведомления»
- Список отправленных SMS-уведомлений по этому заказу (дата, текст, статус доставки, если API возвращает)
- Кнопка «Отправить SMS повторно» (для администратора)

### 4.3 Блок быстрого изменения статуса
- Выпадающий список доступных статусов (ограничен правилами переходов и ролью пользователя)
- Поле для комментария (обязательное при некоторых переходах, например, при отказе клиента)
- Кнопка «Изменить статус»
- При переходе в статус «Готов» автоматически отправляется SMS-уведомление клиенту (с использованием шаблона из настроек).

### 4.4 Блок итогов
- Отображается окончательная стоимость (если заполнена)
- Кнопка «Завершить заказ» (для администратора) — переводит в статус «Выдан», фиксирует окончательную стоимость, закрывает заказ для дальнейших изменений.

---

## 5. Массовые операции (для администратора)

- Возможность отметить несколько заказов чекбоксами в списке и выполнить:
  - **Изменение статуса** (например, все выбранные заказы перевести в «Готов»)
  - **Назначить мастера** (для заказов в статусе «Принят» или «Диагностика»)
  - **Экспорт выбранных** в XLSX
- Все массовые операции логируются.

## 6. Права доступа к функциям экрана «Заказы»

| Действие | Администратор | Менеджер | Мастер |
|----------|---------------|----------|--------|
| Просмотр списка заказов | ✅ все | ✅ все | ✅ только свои (назначенные) |
| Создание заказа | ✅ | ✅ | ❌ |
| Редактирование заказа (все поля) | ✅ | ❌ (кроме комментария, стоимости, статусов до «Согласование») | ❌ (только статус и материалы) |
| Изменение статуса | ✅ все переходы | ✅ до «Согласование», «Готов» | ✅ по своему заказу: Диагностика → Согласование → Ожидание запчастей → Ремонт → Готов |
| Добавление материалов | ✅ | ❌ | ✅ (к своим заказам) |
| Просмотр истории | ✅ | ✅ | ✅ (свои заказы) |
| Отправка SMS повторно | ✅ | ❌ | ❌ |
| Массовые операции | ✅ | ❌ | ❌ |
| Экспорт | ✅ | ✅ (только по своим или с ограничением) | ❌ |

## 8. Технические особенности реализации

- **Синхронизация с БД:** все изменения (статус, материалы) фиксируются в реальном времени через API.
- **Автосохранение:** при редактировании формы есть черновики (необязательно, но полезно).
- **Уведомления:** при изменении статуса заказа (особенно «Готов») отправляется SMS через интегрированный шлюз.
- **Логирование:** каждое изменение заказа записывается в таблицу `order_status_history` с указанием пользователя, времени, нового статуса и комментария.

---

Экран «Заказы» является основным рабочим инструментом для сотрудников «Мастер Принт». Его продуманная структура позволяет быстро находить нужные заказы, оперативно менять статусы и вести учёт материалов, что значительно снижает время обработки и повышает прозрачность работы сервисного центра.
