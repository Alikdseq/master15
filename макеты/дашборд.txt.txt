## Дашборд CRM-системы для «Мастер Принт»

Дашборд — это главный экран, который открывается после авторизации. Он предназначен для быстрого контроля ключевых показателей и оперативного управления. Вид дашборда зависит от роли пользователя.

---

### 1. Общие принципы

- Адаптивный дизайн, хорошо читается на десктопе, планшете и смартфоне.
- Информация обновляется при загрузке страницы и может быть обновлена вручную по кнопке «Обновить».
- Используются графики (Chart.js / D3.js) и цветовая индикация для привлечения внимания к проблемным зонам.
- Ключевые метрики выводятся в виде карточек с крупными цифрами.

---

### 2. Роли пользователей и доступ к дашборду

| Роль | Доступ к дашборду |
|------|------------------|
| **Администратор** | Полный дашборд (все блоки). |
| **Менеджер** | Ограниченный дашборд: общие показатели заказов, список заказов с приоритетом (новые, ожидающие согласования), список товаров ниже порога (только просмотр). |
| **Мастер** | Упрощённый дашборд: список назначенных заказов с фильтром по статусам, индикатор загрузки (количество заказов в работе). |

---

### 3. Блоки дашборда администратора

#### 3.1 Верхняя панель (KPI карточки)

| Показатель | Описание | Источник данных |
|------------|----------|----------------|
| Заказов сегодня | Количество заказов, принятых за текущий день | `orders` с датой приёма = сегодня |
| Заказов за неделю | Количество заказов за последние 7 дней (включая сегодня) | `orders` с датой приёма >= дата-7 |
| Заказов за месяц | Количество заказов за последние 30 дней | `orders` с датой приёма >= дата-30 |
| Выручка за месяц | Сумма `final_cost` по заказам со статусом «Выдан» за последние 30 дней | `orders` с `status='completed'` и датой завершения в периоде |
| Среднее время выполнения | Среднее количество дней от даты приёма до даты выдачи по завершённым заказам за месяц | `orders` с `status='completed'` |
| Товаров ниже минимума | Количество товаров, у которых `current_stock <= min_stock` | `products` с `current_stock <= min_stock` (счётчик) |

#### 3.2 График «Динамика заказов»

- Линейный график, показывающий количество принятых заказов по дням за последние 30 дней.
- Возможность переключения на неделю / месяц / квартал.
- По клику на точку графика можно перейти к списку заказов за этот день.

#### 3.3 Круговая диаграмма «Распределение заказов по статусам»

- Показывает долю заказов в каждом статусе (в процентах и количестве).
- По клику на сегмент можно перейти к списку заказов с этим статусом.

#### 3.4 Таблица «Топ-5 услуг / неисправностей»

- На основе поля `issue_description` (группировка по частоте упоминаний ключевых слов). Рекомендуется использовать предопределённый список категорий неисправностей (например, «не печатает», «шум», «не включается», «плохая печать», «заправка»). Администратор может донастраивать категории в настройках.
- Отображается количество заказов по каждой категории за последние 30 дней.

#### 3.5 Таблица «Товары, требующие заказа»

- Список товаров, у которых `current_stock <= min_stock`.
- Колонки: наименование, артикул, текущий остаток, минимальный остаток.
- Кнопка «Перейти к складу» для быстрого пополнения.

#### 3.6 Список «Срочные заказы»

- Заказы, у которых статус «Ожидание запчастей» > 7 дней или статус «Согласование» > 3 дней (настраиваемый порог).
- Выводятся номер заказа, клиент, устройство, дата приёма, текущий статус.
- Кнопка «Взять в работу» (для мастера) или «Связаться с клиентом» (для менеджера).

#### 3.7 Блок «Активность сотрудников»

- Для администратора: список мастеров с количеством завершённых заказов за неделю, среднее время выполнения.
- Для менеджера: список его текущих заказов, требующих согласования.

#### 3.8 Быстрые действия

- Кнопка «Создать заказ» (открывает форму создания заказа).
- Кнопка «Поступление товара» (открывает форму приходной накладной).
- Кнопка «Сформировать отчёт» (быстрый переход в раздел отчётов).

---

### 4. Дашборд менеджера

- Карточки: «Новые заказы сегодня», «Заказы, ожидающие согласования», «Заказы, готовые к выдаче».
- Список заказов, требующих согласования (статус «Согласование») с кнопкой «Позвонить клиенту» (открывает номер телефона в приложении звонков).
- Список заказов со статусом «Готов» (для контроля выдачи).
- Уведомления о товарах ниже порога (только просмотр).

---

### 5. Дашборд мастера

- Карточка: «В работе сейчас» (количество заказов со статусом «В ремонте», назначенных на мастера).
- Список назначенных заказов с фильтром по статусам (в работе, ожидание запчастей).
- Для каждого заказа: номер, клиент, устройство, статус, кнопка «Изменить статус».
- Кнопка «Завершить ремонт» (переводит заказ в статус «Готов»).

---

### 6. Техническая реализация дашборда

- **API-эндпоинты:**
  - `GET /api/dashboard/stats` – возвращает все карточки KPI.
  - `GET /api/dashboard/charts` – данные для графиков (метки и значения).
  - `GET /api/dashboard/urgent-orders` – список срочных заказов.
  - `GET /api/dashboard/low-stock` – товары ниже порога.
  - `GET /api/dashboard/top-services` – топ услуг.
  - `GET /api/dashboard/activity` – активность сотрудников (для администратора).
- **Периодичность обновления:** данные загружаются один раз при входе; кнопка «Обновить» делает повторные запросы.
- **Кэширование:** для снижения нагрузки агрегированные данные могут кэшироваться на 5-10 минут.

### 8. Рекомендации по дальнейшему развитию

- Добавить виджеты с прогнозами (например, прогноз загрузки мастеров на следующую неделю).
- Интеграция с телеграм-ботом для оперативных уведомлений о ключевых событиях (например, при достижении порога низкого остатка).
- Возможность кастомизации дашборда: администратор может выбирать, какие блоки отображать и в каком порядке.

Дашборд является центральным элементом управления для руководителя и ключевым инструментом для оперативного контроля загрузки мастеров и состояния склада.
