# Финансовый учёт в CRM «Мастер Принт» — принципы расчёта стоимости, затрат и прибыли

Данный раздел описывает логику работы финансового модуля, который позволяет прозрачно учитывать доходы, расходы и прибыль по каждому заказу и в целом по сервисному центру. Основной принцип, заложенный в систему: **стоимость для клиента = стоимость работы + стоимость использованных материалов (по продажным ценам); расходы = себестоимость материалов (закупочная цена) + возможно другие затраты (зарплата, накладные).** Это даёт чёткое понимание реальной прибыльности ремонта.

---

## 1. Основные финансовые сущности

### 1.1 Стоимость работы (труд)
- **Предварительная стоимость работы** (`preliminary_cost`) – указывается при создании заказа или на этапе диагностики.
- **Окончательная стоимость работы** (`final_cost`) – фиксируется при завершении заказа (статус «Готов» или «Выдан»). Может отличаться от предварительной.
- В системе работа и материалы разделены, но для клиента итоговая сумма – это `final_cost` (работа + материалы по продажным ценам).

### 1.2 Стоимость материалов для клиента
- Каждый товар (расходник, запчасть) имеет **продажную цену** (`selling_price`).
- При добавлении материала в заказ:
  - В карточке заказа отображается список использованных материалов с их продажной ценой и количеством.
  - Сумма продажных цен материалов автоматически добавляется к итоговой стоимости заказа (`final_cost`), если администратор/мастер не скорректировал её вручную.

### 1.3 Себестоимость материалов (затраты)
- Каждый товар имеет **закупочную цену** (`purchase_price`).
- При списании материала в заказе фиксируется его закупочная стоимость как расход.
- В финансовых отчётах себестоимость материалов рассчитывается как сумма (`purchase_price` × количество) по всем использованным в заказе товарам.

### 1.4 Дополнительные расходы (опционально)
- Могут добавляться вручную (например, транспортные расходы, аренда, зарплата) через отдельный интерфейс или как накладные расходы на заказ. Для упрощения на первом этапе можно ограничиться только себестоимостью материалов.

## 2. Логика работы с ценами в интерфейсе

### 2.1 Создание / редактирование заказа

**Блок «Стоимость»**:
- Поле **«Предварительная стоимость работы»** (ручной ввод, числовое).
- Поле **«Окончательная стоимость работы»** – до завершения заказа неактивно, заполняется позже.
- При добавлении материалов в заказ (см. ниже) система отображает:
  - **Стоимость материалов для клиента** (сумма `selling_price`).
  - **Себестоимость материалов** (сумма `purchase_price`) – видна только администратору.
- **Итоговая сумма заказа** (к оплате) = `final_cost` (работа) + стоимость материалов для клиента. Это значение можно переопределить вручную (например, если мастер сделал скидку на работу).

### 2.2 Добавление материалов в заказ (для мастера / администратора)

При открытии вкладки «Материалы» в карточке заказа:
- Список доступных товаров (поиск по наименованию/артикулу).
- Добавление товара: выбор количества, автоматическая подстановка продажной цены (можно изменить вручную, если цена договорная).
- После добавления:
  - В карточке заказа обновляется блок «Использованные материалы».
  - Система автоматически:
    - Списывает товар со склада (уменьшает `current_stock`).
    - Создаёт запись в `order_used_products` с указанием `product_id`, `quantity`, а также `selling_price_at_moment` и `purchase_price_at_moment` (чтобы сохранить цены на момент ремонта, даже если позже они изменятся).
    - Пересчитывает итоговую сумму к оплате: `final_cost` (работа) + сумма продажных цен материалов.

### 2.3 Завершение заказа (статус «Готов» / «Выдан»)

- Администратор или менеджер может скорректировать окончательную стоимость работы, если необходимо.
- Если `final_cost` не была указана ранее, система предложит принять автоматически рассчитанную сумму (работа + материалы).
- После подтверждения заказ закрывается, и все финансовые показатели фиксируются.

## 3. Хранение финансовых данных в БД

### 3.1 Таблица `orders` (дополнительные поля)

```sql
ALTER TABLE orders ADD COLUMN final_work_cost NUMERIC; -- окончательная стоимость работы (без материалов)
ALTER TABLE orders ADD COLUMN final_materials_cost NUMERIC; -- сумма продажных цен использованных материалов
ALTER TABLE orders ADD COLUMN total_amount NUMERIC; -- итого к оплате (final_work_cost + final_materials_cost)
ALTER TABLE orders ADD COLUMN materials_cost_price NUMERIC; -- себестоимость использованных материалов (сумма purchase_price)
ALTER TABLE orders ADD COLUMN profit NUMERIC; -- прибыль по заказу (total_amount - materials_cost_price - other_costs)
ALTER TABLE orders ADD COLUMN other_costs NUMERIC DEFAULT 0; -- дополнительные расходы (опционально)
```

### 3.2 Таблица `order_used_products` (снапшот цен)

```sql
CREATE TABLE order_used_products (
    order_id INT REFERENCES orders(id),
    product_id INT REFERENCES products(id),
    quantity INT,
    selling_price_at_moment NUMERIC, -- цена продажи на момент добавления
    purchase_price_at_moment NUMERIC, -- закупочная цена на момент добавления
    PRIMARY KEY (order_id, product_id)
);
```

### 3.3 Таблица `financial_transactions` (опционально)

Для детализации всех финансовых движений можно добавить отдельную таблицу, но для базового учёта достаточно информации в заказах.

## 4. Расчёт прибыли и себестоимости

### 4.1 По заказу

- **Выручка** = `total_amount` (то, что заплатил клиент).
- **Себестоимость материалов** = `materials_cost_price` (сумма закупочных цен).
- **Затраты на работу** (если учитывать зарплату мастера) – можно добавить как `other_costs` или рассчитывать как процент от стоимости работы.
- **Прибыль** = `total_amount` – `materials_cost_price` – (дополнительные расходы).

### 4.2 По складу (маржинальность товара)

- **Маржинальность** = (`selling_price` – `purchase_price`) / `selling_price` × 100%.
- В отчётах можно отображать, какой товар приносит наибольшую прибыль.

### 4.3 Общая отчётность за период

- Суммарная выручка = SUM(`total_amount`) по заказам со статусом «Выдан» за период.
- Суммарные затраты = SUM(`materials_cost_price` + `other_costs`).
- Чистая прибыль = выручка – затраты.

## 5. Интерфейс для финансового учёта

### 5.1 В карточке заказа (для администратора)

Добавляется блок **«Финансы»**, где отображаются:
- Стоимость работы (окончательная)
- Материалы (список с продажными ценами)
- Итого к оплате
- Себестоимость материалов (видна только админу)
- Дополнительные расходы (поле ввода)
- Расчётная прибыль

Пример:

```
┌─────────────────────────────────────────────┐
│ Финансы                                     │
│ Стоимость работы:       2 500 ₽             │
│ Материалы:                                 │
│   Картридж HP 85A (2 шт): 1 200 ₽           │
│   Тонер (1 шт):          800 ₽              │
│ Итого к оплате:          4 500 ₽            │
│ ─────────────────────────────────────────── │
│ Себестоимость материалов: 1 900 ₽           │
│ Доп. расходы:              0 ₽              │
│ Прибыль по заказу:       2 600 ₽            │
└─────────────────────────────────────────────┘
```

### 5.2 В отчётах (вкладка «Финансовый отчёт»)

- Сводка по доходам, расходам, прибыли за период.
- Детализация по заказам.
- Анализ маржинальности товаров.

---

## 6. Варианты гибкости

- **Скидки**: если мастер предоставляет скидку, он может вручную уменьшить `total_amount`, не меняя себестоимость.
- **Бесплатные услуги**: если работа выполнена бесплатно, `final_work_cost = 0`, но материалы всё равно учитываются.
- **Использование материалов, купленных клиентом**: можно добавить товар с нулевой продажной ценой, чтобы он не увеличивал сумму к оплате, но мог быть учтён в расходных материалах (опционально).

---

## 7. Технические рекомендации

- **Триггеры в БД**: при добавлении/удалении материалов автоматически пересчитывать `final_materials_cost`, `total_amount`, `materials_cost_price`, `profit`.
- **Контроль целостности**: не допускать изменения цен на товары, уже использованные в заказах (использовать снапшоты цен в `order_used_products`).
- **Резервное копирование**: финансовые данные критически важны, должны включаться в ежедневные бэкапы.

---


Предложенная модель финансового учёта полностью соответствует требованию заказчика: стоимость материалов суммируется к стоимости работы, а расходы учитываются по себестоимости материалов. Это даёт прозрачную картину реальной прибыли, позволяет анализировать эффективность каждого заказа и принимать обоснованные управленческие решения.
