# Экран «Отчёты» — подробное описание

Модуль «Отчёты» предоставляет руководителю и менеджерам инструменты для анализа деятельности сервисного центра, выявления тенденций, контроля финансов и принятия обоснованных управленческих решений. Все отчёты можно фильтровать по дате и другим параметрам, просматривать в виде таблиц и графиков, а также экспортировать в форматы XLSX и PDF.

---

## 1. Общая структура экрана

Экран «Отчёты» организован в виде вкладок (табов), каждая из которых отвечает за определённый тип отчёта. Доступ к вкладкам зависит от роли пользователя.

### 1.1 Роли и доступ
- **Администратор** – полный доступ ко всем отчётам.
- **Менеджер** – доступ к отчётам по заказам (без финансовой детализации), клиентам (анонимизированная статистика), частично к отчётам по услугам.
- **Мастер** – только отчёт по своей загрузке (если реализован).

### 1.2 Общие элементы управления
- **Период** (быстрый выбор: сегодня, неделя, месяц, квартал, год, произвольный диапазон дат).
- **Кнопка «Сформировать»** – запускает построение отчёта.
- **Кнопка «Экспорт»** – выгрузка в XLSX (табличные данные) или PDF (графики и сводки).
- **Кнопка «Сбросить фильтры»**.

---

## 2. Вкладка «Отчёт по заказам»

### 2.1 Назначение
Анализ объёма работ, эффективности мастеров, динамики заказов и загрузки сервиса.

### 2.2 Параметры фильтрации
- **Период** (обязательный)
- **Статусы заказов** (мультивыбор: все / принят / диагностика / … / выдан)
- **Мастер** (выбор из списка сотрудников)
- **Клиент** (поиск по имени / телефону)
- **Тип устройства** (принтер, ноутбук, МФУ и т.д.)

### 2.3 Выходные данные
Отчёт состоит из двух блоков:

#### 2.3.1 Сводная статистика (карточки)
- Общее количество заказов
- Количество завершённых (статус «Выдан»)
- Количество в работе
- Среднее время выполнения (в днях)
- Общая выручка (сумма `final_cost` по завершённым)
- Средний чек

#### 2.3.2 Динамика заказов (график)
- Линейный или столбчатый график по дням/неделям/месяцам (в зависимости от выбранного периода).
- Возможность отобразить количество заказов и выручку на одном графике (две оси).

#### 2.3.3 Детальная таблица заказов
- Колонки: номер заказа, дата приёма, клиент, устройство, статус, мастер, дата завершения, стоимость.
- Сортировка по любой колонке.
- Поиск по таблице.

#### 2.3.4 Распределение по статусам (круговая диаграмма)
- Доля заказов в каждом статусе за выбранный период.

#### 2.3.5 Эффективность мастеров
- Таблица: мастер, количество принятых заказов, количество завершённых, среднее время выполнения, общая выручка по заказам мастера.
- Ранжирование по количеству или времени.

### 2.4 Экспорт
- В XLSX выгружается сводная статистика и детальная таблица заказов.
- В PDF – сводка и графики.

## 3. Вкладка «Финансовый отчёт»

### 3.1 Назначение
Контроль доходов, расходов и прибыльности сервисного центра.

### 3.2 Параметры фильтрации
- **Период** (обязательный)
- **По заказам** / **По товарам** (переключатель)

### 3.3 Выходные данные

#### 3.3.1 Доходы и расходы (сводка)
- **Доходы**: сумма `final_cost` по завершённым заказам.
- **Расходы** (если ведётся учёт закупочных цен):
  - Сумма закупочной стоимости материалов, списанных в заказах за период.
  - Прочие расходы (можно вводить вручную через настройки, например аренда, зарплата).
- **Прибыль**: доходы – расходы.
- **Рентабельность**: (прибыль / доходы) × 100%.

#### 3.3.2 Детализация по заказам
- Таблица заказов с указанием выручки, себестоимости материалов, прибыли по каждому заказу.
- Группировка по категориям услуг / типам устройств.

#### 3.3.3 Детализация по товарам
- Таблица: товар, количество проданных (списанных) единиц, закупочная стоимость, продажная стоимость, маржинальность.

#### 3.3.4 График динамики выручки и прибыли
- Линейный график по выбранному периоду (дни/недели).

### 3.4 Экспорт
- XLSX с детальными таблицами и сводкой.
- PDF с графиками и сводными цифрами.

---

## 4. Вкладка «Складские отчёты»

### 4.1 Назначение
Анализ остатков, оборачиваемости и движения товаров.

### 4.2 Подвкладки

#### 4.2.1 Текущие остатки
- Таблица всех товаров: артикул, наименование, категория, текущий остаток, минимальный остаток.
- Подсветка товаров ниже порога.
- Возможность указать «остатки на дату» (исторический срез).

#### 4.2.2 Движение товаров
- Параметры: период, товар (можно выбрать конкретный или все).
- Таблица движений: дата, тип (приход/расход), количество, причина (номер заказа, ручное списание), комментарий.
- Итог по количеству поступлений и списаний за период.

#### 4.2.3 Оборачиваемость товаров
- Для выбранного периода: сколько раз товар был списан (использован) относительно среднего остатка.
- Рекомендация: выделить товары с низкой оборачиваемостью (залежавшиеся).

#### 4.2.4 Товары, требующие заказа
- Автоматически формируемый список товаров с `current_stock <= min_stock`, с рекомендуемым количеством для заказа (до `min_stock + 20%`).
- Кнопка «Сформировать заказ поставщику» (экспорт в XLSX).

### 4.3 Экспорт
- XLSX с выбранными данными.
- Для отчёта по заказу поставщику – специальный формат (CSV).

---

## 5. Вкладка «Клиентская аналитика»

### 5.1 Назначение
Анализ клиентской базы, повторных обращений, сегментации.

### 5.2 Параметры фильтрации
- Период (по дате последнего заказа)
- Теги клиентов (мультивыбор)

### 5.3 Выходные данные

#### 5.3.1 Количество клиентов
- Всего клиентов
- Новые клиенты (первое обращение в периоде)
- Активные клиенты (заказы в периоде)
- Вернувшиеся клиенты (повторные обращения)

#### 5.3.2 Повторяемость
- Доля клиентов, сделавших >1 заказа.
- Среднее количество заказов на одного клиента.
- График «Количество заказов на клиента» (гистограмма).

#### 5.3.3 Сегментация по тегам
- Круговая диаграмма распределения клиентов по тегам (VIP, постоянные, новые и т.д.).

#### 5.3.4 Топ-10 клиентов по сумме заказов
- Таблица: клиент, количество заказов, общая сумма, средний чек.

#### 5.3.5 Топ-10 клиентов по частоте обращений
- Аналогично, но по количеству заказов.

### 5.4 Экспорт
- XLSX с полным списком клиентов и их статистикой.
- PDF с графиками.

---

## 6. Вкладка «Отчёт по услугам / неисправностям»

### 6.1 Назначение
Анализ популярности услуг, выявление типовых неисправностей для планирования запасов.

### 6.2 Параметры фильтрации
- Период

### 6.3 Выходные данные

#### 6.3.1 Популярность услуг
- Таблица: категория неисправности (на основе классификации из поля `issue_description`), количество заказов, доля от общего числа.
- Возможность группировки по типу устройства.

#### 6.3.2 Динамика услуг
- Линейный график изменения популярности конкретной услуги по месяцам.

#### 6.3.3 Связь с расходниками
- Для каждой популярной неисправности – какие товары чаще всего использовались при ремонте (полезно для прогнозирования запасов).

### 6.4 Экспорт
- XLSX с детальной таблицей.

---

## 7. Вкладка «Загрузка мастеров» (только администратор)

### 7.1 Назначение
Планирование ресурсов, выявление перегрузок или простоя.

### 7.2 Параметры фильтрации
- Период (по датам приёма)
- Мастер (можно выбрать одного или всех)

### 7.3 Выходные данные

#### 7.3.1 Таблица загрузки
- Мастер, количество заказов в работе (на конец периода), количество завершённых, среднее время выполнения, процент загрузки (относительно среднего по всем).

#### 7.3.2 График «Загрузка по дням»
- Для выбранного мастера – количество заказов, назначенных на каждый день периода (или принятых).

### 7.4 Экспорт
- XLSX.

---

## 8. Вкладка «Кастомные отчёты» (расширение)

Администратор может создавать и сохранять собственные шаблоны отчётов, комбинируя поля из разных модулей (например, «клиенты, которые не обращались более 6 месяцев, и их последнее устройство»). Этот функционал может быть реализован на втором этапе.

---

## 9. Технические детали

### 9.1 API-эндпоинты (примеры)

- `POST /api/reports/orders` – тело запроса: `{ date_from, date_to, statuses, master_id, client_id, device_type }` → возвращает сводку и детальный список.
- `POST /api/reports/finance` – аналогично, возвращает доходы/расходы.
- `POST /api/reports/inventory/current` – текущие остатки (можно с датой).
- `POST /api/reports/inventory/movements` – движения.
- `POST /api/reports/clients` – клиентская статистика.
- `POST /api/reports/services` – анализ услуг.
- `POST /api/reports/master-load` – загрузка мастеров.

Все отчёты должны быть оптимизированы для работы с большими объёмами данных (использовать агрегацию на стороне БД, кэширование).

### 9.2 Хранение шаблонов отчётов
- Настройки отчётов (выбранные фильтры, порядок колонок) могут сохраняться в БД как пользовательские настройки, чтобы менеджеры не настраивали их каждый раз заново.

### 9.3 Фоновое формирование
- Для отчётов с большим объёмом данных (например, за год) можно реализовать фоновую генерацию с уведомлением по готовности.

Модуль «Отчёты» превращает накопленные данные в действенные инсайты, позволяя руководителю принимать решения на основе реальных цифр, а не интуиции. Он обеспечивает прозрачность всех аспектов работы сервисного центра и служит основой для стратегического планирования.
