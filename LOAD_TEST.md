## Нагрузочное тестирование (TC‑46) — порядок прогона

Цель: имитировать **10–20 одновременных сессий** сотрудников и проверить, что типовые API запросы укладываются в **< 500 мс** (ТЗ 7, TC‑46).

### Вариант 1 (рекомендуется): k6

1) Установить k6
2) Поднять backend + postgres + redis
3) Запустить:

```bash
k6 run loadtest/k6_orders.js
```

### Вариант 2: ApacheBench / wrk (если доступно)

Прогонить эндпоинты:
- `GET /api/orders/`
- `GET /api/reports/dashboard/` (admin)

### Что считаем успехом
- p95 latency < 500 ms на типовых GET
- нет 5xx
